## 3. "Wrong layer" findings
*   **Entire FastAPI Backend:** According to the canonical architecture, *all* of `backend/main.py`, `backend/api/*.py`, and related modules are in the wrong layer. They should be migrated to Next.js App Router API routes (`/app/api/v1/...`).
*   **Edge Functions for App Logic:** `track-event` and `send-email` edge functions should be migrated. `track-event` processes async events (like "PayrollProcessed" generating payslips), which belongs in Inngest. `send-email` calls Resend API, which belongs in a Next.js API route or Inngest job calling a `/lib/integrations/resend` wrapper.
*   **AI/ML Logic in Python:** `ai_expense_engine.py` and `ai_recruitment.py` mock or call LLMs. This should be in `/lib/services/ai/*.ts` utilizing `/lib/integrations/...` clients, not a separate Python service.
*   **Scheduling Logic in Python:** `api/scheduling.py` uses OR-Tools. Unless this requires a binary library only available in Python/Go, it needs migration, or if it fits the exception, it's the *only* thing that might remain as a service. However, for now, we assume migration or at least it's a gap.
*   **Tenancy Filters in App Code:** (Assuming from typical FastAPI) FastAPI endpoints likely manually filter by `company_id`. These must be moved to Supabase RLS policies.
*   **Background Jobs in Python:** `auto_gl.py` seems to handle double-entry accounting side-effects. This should be triggered by Postgres outbox + Inngest functions.

## 4. Migration cost estimate
*   **FastAPI to Next.js API Routes:** Large (L) effort. There are 216+ route definitions in FastAPI. This includes Auth, HRMS, CRM, Finance, Assets, AI, etc.
*   **Python AI/ML to TypeScript:** Medium (M) effort. Migrating the mocked or LLM integrations from python to TS.
*   **Python Auto GL / Side-effects to Inngest:** Medium (M) effort.
*   **Edge Functions to Inngest / TS:** Small (S) effort.
*   **Total Estimated Effort:** Extra Large (XL).

## 5. Recommended consolidation plan
1.  **Foundation:** Initialize Next.js App Router structure (`/app/api/v1`, `/lib/services`, `/lib/integrations`, `/lib/db`, `/inngest/functions`).
2.  **Auth & Users:** Migrate `backend/api/index.py` auth routes to Next.js API.
3.  **Core Modules (CRUD):** Migrate simpler modules (CRM, Vendors, Stores, Inventory) from `backend/main.py` to Next.js API.
4.  **Complex Logic & RLS:** Migrate tenancy filters to RLS. Migrate HRMS, Resignations, Assets.
5.  **Side Effects & Inngest:** Migrate `track-event` edge function and `auto_gl.py` logic to Postgres Triggers + Event Outbox + Inngest functions.
6.  **AI Services:** Migrate `ai_expense_engine.py` and `ai_recruitment.py` to `/lib/services/ai/`.
7.  **Integrations:** Migrate `send-email` edge function to `/lib/integrations/resend` and Inngest/API.
8.  **Heavy Processing (Gap Check):** Evaluate `api/scheduling.py` (OR-Tools). If it cannot be ported to TS, document gap for founder review.
